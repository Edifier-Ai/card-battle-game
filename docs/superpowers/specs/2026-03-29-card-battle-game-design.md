# 卡牌对战游戏设计文档

## 项目概述

开发一款类皇室战争的 iOS 实时卡牌对战游戏，使用 Unity 引擎实现 3D 角色在 2D 场景中的混合渲染风格。

### 核心特性
- 实时对战（与 AI 对战）
- 50-60 张卡牌，6 种类型，4 种稀有度
- 免费友好型抽卡系统
- 多难度 + 多性格 AI
- 5 个主题战场，各有轻微机制差异
- 卡牌收集升级系统
- AI 对战 + 练习模式

## 一、整体架构

```
┌─────────────────────────────────────────────────────┐
│                    游戏管理层                         │
│  (Game Manager, Scene Controller, Audio Manager)    │
├─────────────────────────────────────────────────────┤
│                     系统层                           │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌─────────┐ │
│  │ 卡牌系统  │ │ 战斗系统  │ │ AI系统   │ │ 经济系统 │ │
│  └──────────┘ └──────────┘ └──────────┘ └─────────┘ │
├─────────────────────────────────────────────────────┤
│                     数据层                           │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐            │
│  │ CardData │ │ PlayerData│ │ BattleConfig│         │
│  └──────────┘ └──────────┘ └──────────┘            │
```

### 核心模块
- **CardSystem** - 卡牌数据、卡组管理、抽卡逻辑
- **BattleSystem** - 单位生成、移动、攻击、技能、死亡
- **AISystem** - 决策引擎、难度控制、性格策略
- **EconomySystem** - 货币、宝箱、商店、奖励
- **ProgressSystem** - 玩家等级、卡牌升级、解锁

## 二、卡牌系统设计

### 卡牌类型（6种）

| 类型 | 作用 | 示例卡牌 |
|------|------|---------|
| 战士 | 近战单位，肉盾或输出 | 骑士、野蛮人 |
| 射手 | 远程攻击，血量较低 | 弓箭手、法师 |
| 骑兵 | 移动快，冲锋伤害 | 野猪骑士 |
| 法术 | 即时效果，伤害/控制 | 火球、冰冻 |
| 建筑 | 防御或生产单位 | 炮塔、哥布林小屋 |
| 英雄 | 强力单位，高费用 | 巨人、飞龙 |

### 稀有度分级

| 稀有度 | 数量占比 | 获取难度 | 特点 |
|--------|---------|---------|------|
| 普通 | 50% | 容易 | 基础实用 |
| 稀有 | 30% | 中等 | 更强技能 |
| 史诗 | 15% | 较难 | 独特机制 |
| 传奇 | 5% | 稀有 | 强力且独特 |

### 卡牌属性
- **基础属性**：生命值、攻击力、攻击速度、移动速度、攻击范围
- **特殊属性**：护甲、技能、克制关系
- **费用范围**：1-10 费圣水

## 三、战斗系统设计

### 战场布局 - 三线战场

```
┌─────────────────────────────────────┐
│          敌方区域                    │
│   ┌─────┐ ┌─────┐ ┌─────┐          │
│   │敌方塔│ │敌方塔│ │敌方塔│  ← 三座塔 │
│   └─────┘ └─────┘ └─────┘          │
│   |左路|  |中路|  |右路|            │
│   ~~~~~~~~河流/桥梁~~~~~~~~~        │
│   |左路|  |中路|  |右路|            │
│   ┌─────┐ ┌─────┐ ┌─────┐          │
│   │我方塔│ │我方塔│ │我方塔│          │
│   └─────┘ └─────┘ └─────┘          │
└─────────────────────────────────────┘
```

- 三条独立路线，单位只能在本路移动
- 中路更短但更危险
- 策略：分路进攻还是集中一路

### 核心机制

| 机制 | 说明 |
|------|------|
| 圣水系统 | 自然恢复，上限10费，出牌消耗 |
| 出兵 | 拖拽卡牌到己方区域释放单位 |
| 路径寻找 | 单位自动寻路攻击敌方单位和建筑 |
| 胜负条件 | 摧毁主城堡或在时限内摧毁更多塔 |

### 单位行为状态机

```
待机 → 移动 → 攻击 → (技能) → 死亡
       ↑        ↑
       └────────┘
```

### 战斗要素
- **伤害类型**：物理/法术/真实
- **护甲系统**：物理减伤
- **克制关系**：骑兵克射手，射手克战士，战士克骑兵
- **特殊效果**：减速、眩晕、持续伤害、击退

## 四、AI 系统设计

### AI 难度等级

| 难度 | 名称 | 特点 |
|------|------|------|
| 简单 | 新手 | 反应慢，随机出牌，不考虑克制 |
| 普通 | 进阶 | 基本策略，会看费用和时机 |
| 困难 | 专家 | 快速反应，懂得克制和组合 |
| 噩梦 | 大师 | 完美操作，预测玩家行为 |

### AI 性格类型

| 性格 | 出牌风格 | 偏好卡牌 |
|------|---------|---------|
| 进攻型 | 快速施压，多路线进攻 | 低费单位、骑兵 |
| 防守型 | 稳健防守，反击为主 | 建筑、射手、法术 |
| 平衡型 | 攻守兼备，随机应变 | 混合卡组 |
| 控制型 | 法术控场，消耗对手 | 法术、远程单位 |
| 波动型 | 存费爆发，一波压制 | 高费单位、群体卡 |

### AI 决策逻辑

```
每 0.5 秒评估一次：
├── 检查当前局势（各路压力、费用、剩余卡牌）
├── 根据性格选择策略（进攻/防守/存费）
├── 筛选可出卡牌（费用足够、位置合理）
├── 选择最优出牌（克制判断、组合搭配）
└── 执行出牌或等待
```

## 五、经济与抽卡系统设计

### 货币系统

| 货币 | 获取方式 | 用途 |
|------|---------|------|
| 金币 | 对战奖励、任务、宝箱 | 购买卡牌、升级卡牌 |
| 宝石 | 每日登录、成就、升级奖励 | 购买宝箱、加速开箱 |

### 宝箱系统

| 宝箱类型 | 开启时间 | 卡牌数量 | 稀有度加成 |
|---------|---------|---------|----------|
| 木箱 | 即时 | 3张 | 无 |
| 银箱 | 3小时 | 5张 | 少量 |
| 金箱 | 8小时 | 10张 | 中等 |
| 魔法箱 | 12小时 | 15张 | 高 |
| 传奇箱 | 24小时 | 1张 | 保底传奇 |

### 免费友好机制

- 每日免费宝箱（木箱 x2）
- 每4小时可领取免费卡牌
- 对战失败也有奖励（金币/经验）
- 每日任务完成可获金币
- 保底机制：连续50次未出史诗/传奇，下次必出

### 商店系统

- 每日刷新3张卡牌可购买
- 卡牌价格随稀有度递增
- 偶尔出现折扣卡牌

## 六、战场主题设计

### 5个主题战场

| 战场 | 视觉风格 | 机制差异 |
|------|---------|---------|
| 王国平原 | 青草、城堡 | 基础战场，无特殊机制 |
| 灼热沙漠 | 黄沙、绿洲 | 所有单位移速 +10%，法术伤害 +15% |
| 冰霜雪域 | 冰雪、极光 | 所有单位移速 -10%，被攻击有15%几率冻结1秒 |
| 神秘森林 | 密林、迷雾 | 中路有迷雾隐藏单位，需要靠近才可见 |
| 火山熔岩 | 岩浆、硫磺 | 每隔45秒随机一路喷发岩浆，造成范围伤害 |

### 战场解锁

- 王国平原：初始解锁
- 灼热沙漠：玩家等级 5 解锁
- 冰霜雪域：玩家等级 10 解锁
- 神秘森林：玩家等级 15 解锁
- 火山熔岩：玩家等级 20 解锁

### 战场选择

- 练习模式：自由选择
- AI 对战：根据玩家等级随机匹配

## 七、玩家进度系统设计

### 玩家等级

| 等级范围 | 解锁内容 |
|---------|---------|
| 1-5 | 基础卡牌、银箱、王国平原 |
| 6-10 | 稀有卡牌、金箱、灼热沙漠 |
| 11-15 | 史诗卡牌、魔法箱、冰霜雪域 |
| 16-20 | 传奇卡牌、传奇箱、神秘森林 |
| 21+ | 火山熔岩、高级任务、赛季奖励 |

### 卡牌升级系统

| 卡牌等级 | 升级所需数量 | 属性提升 |
|---------|-------------|---------|
| 1级 → 2级 | 2张相同卡 | +10% |
| 2级 → 3级 | 4张相同卡 | +15% |
| 3级 → 4级 | 8张相同卡 | +20% |
| 4级 → 5级 | 16张相同卡 | +25% |
| 5级 → 最高 | 32张相同卡 | +30% |

### 卡组系统

- 初始卡组槽位：3个
- 玩家等级每提升5级解锁1个额外槽位
- 最高可拥有6个卡组

### 成就系统

| 成就类型 | 示例 |
|---------|------|
| 对战成就 | 首胜、连胜、摧毁X座塔 |
| 收集成就 | 收集X张卡牌、满级卡牌 |
| 特殊成就 | 使用特定卡牌获胜、完成特殊操作 |

## 八、分阶段开发计划

### Phase 1 - MVP（核心可玩）

**目标**：实现最小可玩版本，验证核心乐趣

**范围：**
- 战斗系统：三线战场、圣水、出兵、单位AI、胜负判定
- 卡牌系统：8张基础卡牌（战士x2、射手x2、骑兵x1、法术x2、建筑x1）
- AI系统：1个基础AI（简单难度）
- 战场：仅王国平原
- UI：主菜单、战斗界面、卡组编辑

**产出**：可对战的游戏原型

---

### Phase 2 - 卡牌与经济

**范围：**
- 扩展至30张卡牌
- 稀有度系统（普通、稀有）
- 宝箱系统（木、银、金）
- 抽卡逻辑
- 玩家等级、卡牌升级
- 商店基础版

---

### Phase 3 - 完整系统

**范围：**
- 扩展至50-60张卡牌（含史诗、传奇）
- 完整AI系统（4难度 + 5性格）
- 5个主题战场
- 练习模式
- 成就系统

---

### Phase 4 - 优化与平衡

**范围：**
- 数值平衡调整
- 性能优化
- 音效与特效
- 教程优化
- Bug修复

## 技术选型

- **游戏引擎**：Unity（免费版 Personal）
- **开发语言**：C#
- **目标平台**：iOS
- **渲染风格**：3D 角色 + 2D 场景混合

---

## 设计决策记录

1. 选择 Unity 而非原生 iOS 框架，因为更适合实现混合渲染风格
2. 选择三线战场而非传统皇室战争布局，增加分路策略深度
3. 选择免费友好型经济，让玩家能通过游玩获得大部分卡牌
4. 选择分阶段迭代开发，快速验证核心玩法
